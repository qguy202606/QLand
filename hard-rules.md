# QLand 工作硬規則

## 1. 備份規則（BACKUP BEFORE CHANGE）

在對任何檔案執行修改、重構、補丁、批次更新之前，**必須先備份**。

### 備份步驟
1. 確認目標檔案或目錄
2. 使用復原點的方式建立備份
3. Commit/push 前確認備份檔存在

## 2. QA 規則（QUALITY ASSURANCE）

所有修改完成後，發布到 Production 前，**必須執行 QA 檢查**。

### QA 檢查項目
1. 無 broken link
2. 無英文字混雜在中文內容中
3. 全為原創繁體中文敘事
4. UI 渲染正常（HTML/CSS 語法正確，無格式崩壞）
5. 章節導航正常（上一章/下一章按鈕對應正確）
6. 系列命名一致（不可擅自更名）

### QA 通過後才能 push

## 3. 其他規則

- 使用者對重複失敗容忍度低：失敗時會明確表達不滿；應先停、診斷、單一路徑修復
- 使用者說「爆爆爆」時代表要快、要直接做、不要過多解釋
- 所有回覆一律使用繁體中文，不得切換英文或其他語言
