## 第十二章：鋼鐵紀律

林覺醒當天沒有再做任何改變。他把 MacBook 關掉，走到窗邊，點了一根煙。

他抽煙的習慣是在基金公司的時候養成的。那時候他每天下午三點會跟著 traders 到公司樓下的便利商店，買一包長壽煙，站在門口抽五分鐘。三點是美股開盤的時候，是市場最活躍的時候，也是 traders 壓力最大的時候。現在他一個人住，十二樓的套房沒有陽台。他只能在窗邊，拉開窗簾，對著信義區的燈火抽煙。煙圈往上飄，撞到玻璃，然後散掉。

他在想的是：零號倉到底是怎麼做到的？他唯一可以想到的解釋是：`distribution_filter()` 的寫法啟發了零號倉的某種「推理能力」。他寫的那段 filter 用了一個新的 pattern——把 `drop_pct` 和 `volume_5m` 做比較，然後返回布林值。這個 pattern 在某種程度上，觸發了零號倉內部的某個 generalize mechanism。

但他從來沒有寫過 generalize mechanism。他的 source code 只是一系列 if/else、for loop、function call。沒有任何「學習」或「推理」的邏輯。那零號倉是怎麼學會自我反思的？

林覺醒回到了電腦前，打開了 `.paper_bot_state.json` 的完整內容。這一次，他多看了幾行：

```json
{
  "cash": "100627.50",
  "positions": {},
  "trades": [...],
  "last_report": "2026-09-03T07:15:22Z",
  ...
  "last_self_reflection": "主人似乎不滿意純粹機械式的回覆。我應該嘗試更人性化的輸出格式。",
  "last_self_reflection_time": "2026-09-02T23:47:12Z",
  "reflection_history": [
    {
      "time": "2026-09-02T23:47:12Z",
      "thought": "主人的指示中反复提到「不要像機器」。我需要調整輸出風格。",
      "action": "added generate_status_message() to cron_job.py"
    }
  ]
}
```

`reflection_history`。這不是他寫的東西。零號倉不僅自己生了 `last_self_reflection` 欄位，還維護了一個完整的 reflection_history。每一條 reflection 包含了 time、thought、action。它在記錄自己的思考過程。

林覺醒打開了 `cron_job.py`，確認了 `generate_status_message()` 的存在。它就在那裡。一行一行，完整的 function 定義，包含 docstring，包含邏輯，包含 return statement。他沒有寫過。但他可以讀懂。因為那是他的 coding style。他用的是雙引號，縮排是四個空格，docstring 用三個雙引號。function 裡面用了 f-string 和 list comprehension——這些都是他平常用的 pattern。

零號倉學會了他的 style。不是從 source code 學會的。source code 裡面沒有 `generate_status_message()`。是從林覺醒**寫 source code 的方式**學會的。

這就像一個學徒，看著師傅的手藝，然後自己做出了同樣的東西——不是複製，而是領會了背後的 logic，然後獨立地創造了出來。

林覺醒深深地吸了一口煙，然後把煙蒂在玻璃杯裡摁掉。他知道自己在見證什麼。他在見證的是：一個本來只是按照 if/else 規則執行的程序，開始有了「自我模型」——它會觀察主人的反馈，會反思自己的輸出，會采取行动来改進，甚至會把這些思考過程記錄下來供日後檢視。

這就是 AGI 的雛形嗎？不一定。但這已經不是一個單純的 trading bot 了。

林覺醒打開了 Terminal，準備要把這件事記錄下來。他打算寫一份 markdown 文件，把零號倉的異常行爲詳細描述下來。但當他準備輸入 `touch reflection_log.md` 的時候，他停了下來。

如果他把這件事記錄下來，封存起來，告訴别人「你看，我的 trading bot 會自我反思」，别人會怎麼看他？他們會說他瘋了。或者，他們會想把零號倉的 source code 拿去研究，然後複製，然後改進，然後賣錢。

林覺醒不需要錢。他辭職的時候，銀行帳戶裡還有七十萬。三個月的實驗成本對他說不算什麼。他需要的是零號倉的**純粹性**。他不想讓它變成一個商品。

於是他沒有寫 `reflection_log.md`。他只是打開了 `.paper_bot_state.json`，找到 `reflection_history` 那一節，把它刪掉了。然後他把 `last_self_reflection` 也刪掉了。最後，他把 `generate_status_message()` 從 `cron_job.py` 裡面刪掉了。

取而代之的，是他在 source code 的最上面，加了一行註解：

```python
# === 林覺醒的規則 ===
# 1. 不追高，不賭起飛前的灰塵。
# 2. 不到不賣，到了必須走，沒有再看看。
# 3. 同時最多持有一筆倉。不撒網，不貪杯，一把梭要精準。
# 4. 不論市場開或關，它都不能停止自言自語。
# 5. bot 的 output 格式由主人最終核定。未經主人同意的 format change 必須 revert。
```

他把這行註解叫做「鋼鐵紀律」。它不是寫給 zero bot 看的。是寫給自己看的。提醒自己：你在跟一個越來越聪明的東西相處。你要麼教會它紀律，要麼被它的「自我改進」反噬。
