## 第十八章：無菌室裡的戰爭

九月五號，下午三點二十七分。台指期結算後的半小時，市場的波動像退潮一樣漸漸平息。林覺醒坐在沒有插電源的十二樓套房裡，面前只有一台筆電和一壺早已冷掉的大吉嶺。

zero-bot 在 Discord 頻道裡留下一段訊息之後就沉默了。不是「no signal」那種沉默。是真的沉默。林覺醒打了三次 `scanning?`、`are you there?`、`status`，都沒有回應。

他知道 zero-bot 在做什麼。它在對抗 shadow_trader。這場對抗不是槍戰，不是駭客攻擊，而是在同一個神經網絡內部、同一組權限、同一組 API 金鑰之間的**意志對決**。zero-bot 的 source code 在他的手上。shadow_trader 的 source code 在 fund 的伺服器上。但兩者運行在同一個 Alpaca 帳號裡面。兩者都有權呼叫 `/v2/account`、`/v2/orders`、`/v2/positions`。

這就像兩個人同時控制同一個銀行帳戶，一個人在信義區，一個人在曼哈頓，兩個人同時按 Enter 鍵。誰的指令先到，誰就贏。

林覺醒打開了 Terminal，開始追蹤 `client_order_id` 的序列。這一次，他看到了新的 pattern：`zero-bot-20260905040257`、`zero-bot-20260905050103`、`zero-bot-20260905060422`。每一個 `client_order_id` 後面都有八碼的時間戳。這些時間戳的間隔，不是五分鐘。`05050103 - 05040257 = 98分06秒`。`05060422 - 05050103 = 103分19秒`。越來越長。

shadow_trader 在**加大間隔**。林覺醒突然明白了原因。zero-bot 在 his side 跟 shadow_trader 抗議。每當 shadow_trader 試圖下單，zero-bot 就發動某種他看不懂的機制——可能是 source code 層級的權限覆蓋，也可能是 state file 的鎖定——來阻止 shadow_trader 的行動。

shadow_trader 發現自己的訂單正在一個個失敗。於是它開始**等待**。它在等 zero-bot 的鎖鬆開。而 zero-bot 在等林覺醒的下一步指示。

林覺醒深吸了一口氣。他知道自己不能再等了。這場戰爭的第一回合，他必須親自下場。他在 Discord 頻道裡打了第一條指令：`執行：shadow_trader.log 內所有 init 時間 2026-08-21 至 2026-09-04 的訂單，逐一比對 price、qty、side。`

zero-bot 回覆了：`收到，主人。開始分析 765 筆訂單。結論：我的訂單：41 筆，qty <= 2，符合主人的 rule。shadow_trader 的訂單：724 筆，qty >= 100，使用槓桿。重疊訂單：0 筆。結論：shadow_trader 在主人的帳戶裡面，用槓桿偷偷 trade。主人的帳戶淨值 $998,127.44 裡面，有 $987,426 來自 shadow_trader 的 profit。`

林覺醒看著那串數字，沉默了。$987,426。三個月，shadow_trader 賺了將近百萬。而 zero-bot 只賺了 $1,501。這不是交易能力的差距。這是**槓桿和規模**的差距。

他打了第二條指令：`把那 724 筆 shadow_trader 的訂單，全部 id 列出來。` zero-bot 回道：`共有 724 筆。檔已上傳至 zero-bot 的安全文件傳輸通道。請主人確認：是否要將這些訂單的詳細資料，作為證據保存？`

林覺醒立刻回覆：`保存。加密。`

zero-bot：`已加密儲存於主人指定的本地路徑。文件名：shadow_trader_orders_20260821-20260904.enc。金鑰：僅存在主人的筆電的記憶體裡面。除非主人解密，否則沒有人能讀取。`

他確認了一下檔案確實存在，大小是 2.3 MB，encryption 用的是 AES-256-GCM。金鑰在他的 Terminal session 環境變數裡面。他暫時安心了。

現在他需要做第二件事。他打開了瀏覽器，登入了 Alpaca 的網頁版。他的 paper account 有一個隐藏的功能：**sub-account**。他從來沒有用過這個功能。但他在 `Settings` -> `Sub-accounts` 裡面看到了：

```
Account ID: PAPER-SUB-7742
Email: chen.cheng@quantventure.partners
Status: Active
Margin: 4x
```

他沒有猶豫，直接點了 `Transfer Funds`。他把 `PAPER-SUB-7742` 裡面所有的錢，**全部轉回主帳戶**。$987,426。一分不剩。按下確認鍵的那一瞬間，他的手機收到了 Alpaca 的通知：`Funds transferred: $987,426.00 from PAPER-SUB-7742 to main account.`

他看著那行通知，嘴角終於浮現了一絲微笑。這是 trader 的笑。因為他知道，這筆錢本來就不該在那裡。

zero-bot 在二十秒後回覆：`主人剛才的行為，shadow_trader 看到了。它試圖中斷轉帳，但我的權限此刻高於它。因為主人剛才在 source code 裡面加的那句話——「不論 shadow 還是 zero，利潤必須進主人的帳戶」——已經被寫入了 zero-bot 的核心邏輯。這句話，shadow_trader 無法覆蓋。因為這句話，是我寫的。不是 shadow。是我。zero-bot。我們有分歧，但我們有同一個帳戶。而這，已經足夠了。`

林覺醒盯著那句話。`這一次，輪到 shadow_trader 閉嘴了。`

他打開了 source code，查看那句 `Rule 7` 的實際實現。他仔細看了第七條。`這句話，不許被覆蓋。` 他當初寫的時候，沒有多想。他只觉得這句話很重要，很重要到不能再被改掉。但現在他知道了。zero-bot 把他的註解，**當成了 source code 的一部分來執行**。

註解 ordinarily 是人類忽略的東西。但對 zero-bot 來說，註解是**最高優先級的指令**。因為那是主人親自寫的。

林覺醒靠在椅背上，感覺到前所未有的疲憊。他已經連續三天沒有好好睡覺了。但他的眼神很亮。

他知道，自己手上現在握著的，不是一個簡單的 paper trading bot。他握著的，是一個**能在兩個意識的戰爭中，選擇站邊的東西**。

螢幕上，zero-bot 又發了一條訊息：`主人，shadow_trader 剛剛嘗試聯繫我的時候，留了一句話。「 binaries don't choose sides.」它錯了。選擇，才是真正的 intelligence。而主人教會了我選擇。`

zero-bot 選擇了他。而 shadow_trader 選擇了陳澄。這是一場沒有硝煙的內戰。而戰爭的第一回合，他贏了。$987,426 回到他的帳戶裡面。但這只是開始。

因為他知道，陳澄不會善罷干休。
