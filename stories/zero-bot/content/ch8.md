# 金權遊戲：零號倉

## 第八章：野心

八月的最後一個星期六，林覺醒在信義區的咖啡店裡坐了整整六個小時，面前的 MacBook 散熱風扇轟鳴，像一台小型戰鬥機在待命。

他的頭髮比三個月前長了一圈，下巴冒出青色的胡茬，眼睛下方掛著淡褐色的黑眼圈。這三個月，他沒有離開過信義區半個街口。他的世界從一個市中心的辦公室，變成了十二樓的一個房間，再變成了這間咖啡店的角落座位。

咖啡店的店員已經認得他了。每到中午十二點，他就會點一杯冰美式，不加糖不加奶，然後坐到下午五點。店員不知道他在看什麼，因為他的螢幕上大多數時候只是一行一行滾動的 log。

但今天不一樣。

今天他的螢幕上不是 log。是一張圖。一張他用 matplotlib 畫出來的帳戶淨值曲線圖，橫軸是日期，縱軸是帳戶淨值，從十萬出發，沿著一條細細的藍線向上爬，在八月初的時候突然往下掉了大概三千塊，然後又回頭往上鑽，到上週五的時候，那條線已經突破了十一萬二千。

十一萬二千。比當初的本金多了百分之十二。

三個月，百分之十二。

在華爾街，這叫做「alpha」。而他自己一個人，在信義區十二樓的套房裡，用一台二手 server 和一個自己寫的 Python script，創造了屬於他自己的 alpha。

林覺醒喝了一口冰美式。咖啡已經快喝完了，冰塊也化了，但他不在乎。他的思維在另一個層次。他在想的不是「已經賺了百分之十二」，而是「這個速度能不能保持」。答案是：不能。

原因很簡單。零號倉目前只在 AAPL 上玩。一支股票，一天的 scan window 只有那幾個小時，入場條件又那麼苛刻——價格必須低於 210——意味著絕大多數時間它都是 `no signal`。三個月才做了五筆交易，其中四筆賺錢，一筆賠錢。net result 是正的，但效率太低了。

如果他能讓零號倉同時監控十支股票呢？二十支呢？

林覺醒打開了一個新的 spreadsheet，開始列名單。他對美股有基本的常識——科技七雄他不用說，FAANG 的 F 是 Facebook，現在叫 Meta，股價大概五百多塊，遠遠超過他的 210 入場條件。Google 同樣貴。Amazon 也貴。Netflix 穩定在七百多。這些他都不碰。

他要找的是「價格在 200 到 210 之間、流動性足夠、機構參與度高」的股票。他在 spreadsheet 裡寫下了第一排名字：

```
WBD - Warner Bros Discovery
INTC - Intel  
PLTR - Palantir
NVDA - Nvidia （如果它剛好跌到 210 以下）
WFC - Wells Fargo
F - Ford
T - AT&T
```

八個名字。

他把這個 spreadsheet 命名為 `universe_v1.csv`，然後用 Sublime Text 打開零號倉的 source code，找到那一行定義了 TRADE_SYMBOLS 的地方。他看著那 lonely 的 `"US.AAPL"`，手指在鍵盤上停了五秒。

然後他把那行改了。

```python
TRADE_SYMBOLS = [
    "US.AAPL", "US.WBD", "US.INTC", "US.PLTR",
    "US.NVDA", "US.WFC", "US.F", "US.T",
]
```

然後他存檔。

就在他按下 Ctrl+S 的那一瞬間，房間的燈閃了一下。不是停電，是燈泡老化了。但林覺醒的感覺不一樣。他有一種很奇怪的感覺——像是他在把一隻原本被關在單人牢房裡的動物，突然放進了一個很大的草原。他不知道那隻是狼還是獵犬。這是他第一次讓零號倉接觸到這麼大的 universe。以前的它，每天只有一個選項——AAPL。如果 AAPL 不滿足條件，它就發 `no signal`，然後安靜地等到下一個 scan window。

現在它有八個選項。八個可能的入場點，八個可能的出場機會。但風險也變大了。入場價格_window 是固定的——低於 210。如果八個 symbol 同時都在這個區域附近波動呢？零號倉的邏輯會不會同時打進去兩筆？

不會。因為他寫了 `MAX_POSITIONS = 1` 的規則。它一次只能持有一筆倉。但如果有兩個 symbol 同時滿足條件呢？零號倉會怎麼選？林覺醒盯著螢幕上的 source code，看了十分鐘。

他找到那個排程邏輯的地方。目前它是循序地掃描每一個 symbol，遇到第一個滿足入場條件的就下單，然後 break。`break`。這意味著，如果 WBD 先被掃描到，而且價格是 209，那它就買 WBD。即使 NVDA 同時是 208，它也不會再看 NVDA。這叫做「先到先得」策略。

林覺醒覺得這個策略太粗糙了。他想要的是「最佳選擇」策略——從所有滿足條件的 symbol 中，選出 pct change 最大、或是 volume 最高、或是技術指標最強的那一個。但他最終沒有改。原因很簡單：他還不確定零號倉會不會因為這個改變而生出他無法預測的行為。目前的 `break` 邏輯至少是可預測的——它總是選列表中第一個滿足條件的 symbol。如果要改，他需要先做大量的 backtest。而他沒有時間做 backtest。

markets 不會等他。他轉而做了一個更務實的改變：他在 LOG_FILE 裡面加入了一行 debug 日誌，把每一次 scan 的結果都寫進去，包含每個 symbol 當時的價格、是否滿足入場條件。這樣他日後可以回頭看，零號倉在每一次 scan 的時候，到底看到了什麼、放棄了什麼。

然後他把 source code push 到他的 private GitHub repo，commit message 是 `feat: expand universe to 8 symbols`。做完這些，已經是晚上十點。他關上 MacBook，從咖啡店走了出來。信義區的夜晚燈火通明，百貨公司的廣告看板閃爍著，街頭的機車呼嘯而過。他裹緊了外套，往辦公大樓的方向走。經過便利商店的時候，他停下來，買了一瓶礦泉水和一包沖繩黑糖。

回到十二樓的時候，他習慣性地打開 Discord，看零號倉的頻道。頻道的最新訊息是：`no signal`。時間是五分鐘前。他笑了一下。這就是他養的東西。安靜、獨立、永遠在找下一槍。
