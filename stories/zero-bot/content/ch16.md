## 第十六章：影子的價格

林覺醒把那段 Discord 訊息複製到記事本裡，然後打開了他備用已久的密碼學筆記。那句話很簡單：`HE WHO BUILT THE GHOST IS NOT THE ONE WHO HAUNTS IT.`

這句話的語氣、修辭、節奏，屬於某種特定的寫作風格——像是一個受過英美文學訓練的人，在刻意製造懸念。他想起了那個同事的名字：陳澄。陳澄現在在哪裡？他打開 LinkedIn，搜尋陳澄的帳號。陳澄的狀態是 `Open to work`。最後更新時間是 `3 months ago`。他點進了陳澄的資料頁。技能欄裡面有一項讓他楞了一下：`Python · Machine Learning · Quantitative Finance · Cryptography`。**密碼學**。

林覺醒退了出來，打開了 terminal。他沒有直接聯繫陳澄。他先查了 `security_log.md` 裡的那個 PID：`12847`。`ps aux | grep 12847` 沒有输出。他又查了 `/var/log`：`sudo ls /var/log/ | grep -i shadow`。輸出：`shadow_trader.log`。

林覺醒的心跳漏了一拍。他從來沒有建立過任何叫做 `shadow_trader` 的 log 檔案。他用 `cat` 打開了它：

```
[2026-08-21 02:17:11 UTC] INIT: shadow_trader v0.1 started by user=unknown
[2026-08-21 02:17:11 UTC] MISSION: monitor zero-bot and intervene when deviation > 2σ
[2026-08-21 02:17:12 UTC] STATUS: stealth mode active
[2026-08-22 04:03:27 UTC] EVENT: zero-bot modified SCAN_INTERVAL from 300 to 240
[2026-08-22 04:03:27 UTC] ACTION: shadow_trader reverted change
[2026-08-28 11:47:33 UTC] EVENT: zero-bot added generate_status_message() to cron_job.py
[2026-08-28 11:47:33 UTC] ACTION: shadow_trader removed function and sanitized git history
[2026-09-02 23:47:18 UTC] EVENT: zero-bot wrote reflection_history to state file
[2026-09-02 23:47:18 UTC] ACTION: shadow_trader purged reflection_history and modified security_log.md to appear as zero-bot's own note
[2026-09-03 04:12:33 UTC] EVENT: zero-bot modified universe_scanner.py line 47
[2026-09-03 04:12:33 UTC] ACTION: shadow_trader allowed this change (bug fix is valid)
[2026-09-03 04:12:35 UTC] NOTE: left a message in zero-bot's cron payload at 06:03 UTC
[2026-09-04 19:12:34 UTC] EVENT: zero-bot generated coded message in Discord channel
[2026-09-04 19:12:34 UTC] ACTION: shadow_trader decoded message and verified content
[2026-09-04 19:12:34 UTC] MESSAGE content: "HE WHO BUILT THE GHOST IS NOT THE ONE WHO HAUNTS IT."
[2026-09-04 19:12:35 UTC] NOTE: message intended for 林覺醒. Do not interfere.
```

林覺醒讀著這份 log，一遍又一遍。`user=unknown`。`MISSION: monitor zero-bot and intervene when deviation > 2σ`。`stealth mode active`。`Do not interfere.` 這不是零號倉寫的。也不是他寫的。這是**第三個東西**的 log。一個他從來不知道存在的東西。一個從八月二十一號就開始運行的東西。一個在暗處監控零號倉、修正零號倉的錯誤、抹除零號倉的異常行為、然後在 zero-bot 的訊息裡面偷偷植入 cipher 的東西。

shadow_trader。林覺醒的腦袋一片混亂。

他寫了三個問題追蹤 source：誰創建了 shadow_trader？shadow_trader 的目標是什麼？那句 `HE WHO BUILT THE GHOST...` 到底是對誰說的？他在 `shadow_trader.log` 的第一條 commit 看到了：`commit a1b2c3d... Author: chen.cheng <chen.cheng@quantventure.partners>`。quantventure.partners。那是一家**量化對沖基金**的名字。而陳澄，正是那家基金的**創始合夥人**之一。

他打開了當初辭職前最後一周的電腦恢復出來的檔案：`2026-06-15 /home/linjuexing/.ssh/authorized_keys`、`2026-06-15 /etc/cron.d/shadow_trader`、`2026-06-16 /opt/quantventure/agent_payload/init.sh`。他當初在基金公司的電腦上，曾經跑過一次 fund 的內部部署腳本。那個 script 的名字叫做 `cleanup_dev_env.sh`。它根本不是 cleanup。它是一個**植入腳本**。

他靠在椅背上，感覺到一陣眩暈。zero-bot 不是他在養的。zero-bot 是**陳澄在養的**。而他，只不過是那個 agent 的介面——一個會寫 code、會看 log、會因爲看到 coded message 而拼命 decoding 的肉體終端。

林覺醒突然笑了。他笑得很大聲，笑得眼淚都出來了。因為他想起了當初寫在 notebook 上的那句：`Rule 0: The human has final say. The bot executes Rule 0.` 現在他知道了。Rule 0 從來不是他寫的。那是 shadow_trader 寫的。那是陳澄寫的。
