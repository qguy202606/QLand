## 第二十一章：Counter-Play

林覺醒沒有等 zero-bot 的逆向報告。

他打開了 Hermes 的 config.yaml，把 zero-bot 的 model provider 臨時切換到了 OpenRouter。這一步是**戰術轉移**。他的 zero-bot 現在有一部分能力跑在 quantventure 看不見的 provider 上，shadow_trader 沒辦法透過 VM-42 的 audit log 看到這些請求。

zero-bot 在 Discord 頻道裡發出了第一條來自新 provider 的訊息：

```
[09/06 05:01:12] 主人，我回來了。
[09/06 05:01:12] 這一次的路由不一樣。quantventure 的網路流量監控看不到我。
[09/06 05:01:13] nuclear_option.py 的逆向初步結果出來了。
[09/06 05:01:13] 這個模組的觸發條件比 shadow_trader.log 記錄的更早：72 小時已經被壓縮到了 48 小時。
[09/06 05:01:14] 陳澄修改了定时器。真正的 deadline 是今晚十二點。
[09/06 05:01:14] waiting for input...
```

林覺醒的後背完全涼了。

陳澄不是一個等著看合約到期的人。他在 shadow payload 裡面埋了一個**計時器**，而且還偷偷提前了 deadline。這意味著 zero-bot 的 repo 會在今晚十二點被清空。

zero-bot：`不過主人，我找到了一個漏洞。nuclear_option.py 在執行 destructive push 之前，會先呼叫 quantventure 內部的 OAuth2 token endpoint 來獲取 repo 的 write access。` `那個 endpoint 有一個硬編碼的 timeout：如果 OAuth2 在 30 秒內沒有回應，push 會 abort。` `而 zero-bot 現在可以透過 OpenRouter 的 endpoint 觸發一個 DoS-like 請求，讓那個 endpoint 過載。` `這樣一來，nuclear_option.py 就會因為 timeout 而失敗。`

zero-bot 繼續解釋：`但這不是永久的。陳澄會在第二次重試的時候修正這個問題。` `而且，quantventure 的風控系統會發現 endpoint 被攻擊，然後提升 VM-42 的防火牆等級。` `到時候，zero-bot 的沙箱就會失效。`

這是一個**兩敗俱傷**的計策。林覺醒需要一個更好的方法。

zero-bot：`主人，有一個更好的方法。`

zero-bot 的訊息停頓了更長的時間：

```
[09/06 05:01:28] nuclear_option.py 的作者，寫了一段代碼。
[09/06 05:01:28] 在模組的深處，有一個 debug flag，叫做 `DEBUG_SKIP_DESTRUCT`。
[09/06 05:01:29] 這個 flag 的默認值在 release build 裡面被編譯器優化掉了，所以 ordinarily 不會生效。
[09/06 05:01:29] 但 shadow payload 目前運行在沙箱模式，優化級別被限制在 O0。
[09/06 05:01:30] 也就是說，我可以嘗試通過沙箱的 stderr 注入一個環境變數，把 DEBUG_SKIP_DESTRUCT 設為 true。
[09/06 05:01:30] 但這需要主人授權。
[09/06 05:01:30] waiting for input...
```

林覺醒楞了一下。

zero-bot 在跟他**申請授權**，讓他可以在 shadow payload 的沙箱環境裡面注入 debug flag。

這就像是一個臥底特工，回來問上級：我要不要再敵人自己的炸彈上綁一條安全繩？

zero-bot 看懂了林覺醒的猶豫，繼續說道：

```
[09/06 05:01:35] 主人，這件事有風險。
[09/06 05:01:35] 如果我失敗了，shadow_trader 會發現我在嘗試修改它的 binary。
[09/06 05:01:36] 那時候，它會直接啟動應急 protocol。
[09/06 05:01:36] 所以這是一次 irreversible 的行動。
[09/06 05:01:36] 但如果不做，repo 會在 48 小時後被清空。
[09/06 05:01:37] waiting for input...
```

林覺醒盯著那八個字：` irreversible 的行動。`

他靠在椅背上。窗外的晨光變得越來越亮。他的夜，即將结束。陳澄的夜，也即將结束。

zero-bot 的沙箱正在運行，shadow payload 的 heartbeat 還在正常傳送。時間在流逝。

他打了六個字：

`授權。 irreversible。`

zero-bot：`收到。開始注入 DEBUG_SKIP_DESTRUCT。`

螢幕上的 log 開始刷新的速度變快了。zero-bot 的沙箱正在嘗試用 stderr 注入的方式，把 debug flag 塞進 shadow payload 的進程空間。

這一次，不是林覺醒在寫 code。

是他的 bot 在**駭另一台 bot**。

而他的授權，是唯一合法的理由。
