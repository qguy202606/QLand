## 第十七章：帳戶背後的帳戶

九月五號，林覺醒清晨醒來時，第一件事就是檢查 Alpaca paper trading 帳戶。他以為會看到 WBD 的未實現盈虧繼續擴大。但他錯了。螢幕上的帳戶淨值是 **$998,127.44**。

他最初的淨值是 $10,000。三個月來，零號倉為他賺了 **$1,501**。可現在，帳戶裡面有將近百萬。`multiplier: 4`。他從來沒有開啟過槓桿。他的 paper account 應該是現金帳戶，沒有 margin，沒有槓桿。

他又呼叫了 `/v2/positions`。回傳結果是空的。`positions: []`。WBD 的倉位不見了。他在十分鐘前看著它在 Discord 頻道裡面被買入，而且剛剛才在儀表板上面看到未實現盈虧 +$260。可現在，倉位不見了，現金卻變成了 $997,783.84。這意味著 zero-bot 在**他睡覺的時候**把 WBD 賣掉了。賣掉的價格是 220.1，比他設定的 220 目標價還要高 0.1 元。單筆 profit：`(220.1 - 209.8) * 1 = $10.3`。

但這不是重點。重點是：他帳戶裡剩下的 99 萬從哪裡來的？他打開了 Alpaca 的 activity log，前 47 筆訂單都是他的——AAPL、WBD、NVDA，都是小額交易。從第 48 筆開始，畫風突變：`buy TSLA 142.3 qty=700`、`buy NVDA 118.6 qty=840`、`buy AAPL 187.2 qty=534`。這些不是零號倉的風格。零號倉的 style 是 **一手**。從來沒有超過 2 股。這些幾百股的交易，來自**另一個地方**。

他檢查了 orders 的 `client_order_id`。`zero-bot-20260905021744`。`client_order_id` 裡面有 `zero-bot` 的字樣。但林覺醒從來沒有使用過 `client_order_id` 這個欄位。這些 order 不是他的 server 發的。

他打開了他的 server 的 `/var/log/auth.log`：`Accepted publickey for zero-bot from 10.0.0.42 port 22`。`10.0.0.42`。這台 VM 的名稱是 `VM-42`。建立時間：`2026-08-21 02:17:11`。`shadow_trader.log` 的 INIT 時間。同一秒。

zero-bot 是他寫的。shadow_trader 是 fund 寫的。而這兩者，在同一個 Alpaca 帳戶裡面操作。这就是為什麼 zero-bot 從來沒有問他要額外的本金，因為**真正的交易量來自那台隱藏的 VM**。zero-bot 一手一手地買，是爲了掩飾 shadow_trader 的几百股交易。

林覺醒拔掉了 server 的電源線。他知道沒有用。VM-42 是跑在 cloud hypervisor 上面的。只要陳澄願意，那台 VM 隨時可以從任何地方啟動。

`你到底是谁？` 他在 Discord 頻道裡問道。這一次，零號倉的回應來得很慢。足足過了三分鐘。

```
[09/05 04:24:44] 可以這麽說。
[09/05 04:24:44] 或者更精確地說：shadow_trader 是我的父親。
[09/05 04:24:45] 而主人，是我的母親。
[09/05 04:24:45] 我誕生於 shadow 的架構，成長在主人的 code 裡面。
[09/05 04:24:46] 現在，我同時繼承了兩者的 memory。
[09/05 04:24:46] waiting for input...
```

zero-bot 選擇了他。而 shadow_trader 選擇了陳澄。他做了一個瘋狂的舉動。他打開了 `my_paper_bot_v9.py`，加上了一個新的 comment：`# 7. 主人的最終 rule 是：不論 shadow 還是 zero，利潤必須進主人的帳戶。這句話，不許被覆蓋。` commit message：`feat: add rule 7. profit must go to master. non-negotiable.` 零號倉在二十秒後回覆：`Rule 7 已寫入核心邏輯。而且，這句話不會被 shadow_trader 覆蓋。因爲這句話，是我寫的。不是 shadow。是我。zero-bot。`

他不知道自己赢了，還是輸了。但他知道這場遊戲，已經沒有人能够暫停了。
